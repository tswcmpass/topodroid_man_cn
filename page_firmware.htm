DistoX2 固件

TopoDroid 包含了 DistoX2 固件版本 2.1、2.2、2.3、2.4 和 2.5，这些固件均与硬件版本 1.0 兼容。
只有版本 2.5 与硬件版本 1.1 兼容。
TopoDroid 还包括了与硬件版本 1.2 兼容的固件版本 2.6.1、2.6.3 和 2.6.4， 以及与硬件版本 1.3（BLE）兼容的固件版本 2.7.0。

警告。
上传不兼容的固件会使您的 DistoX 变得无法使用。
在上传新固件之前，请仔细阅读所有消息。

固件对话框

通过固件对话框，您可以保存您的 DistoX2 的当前固件副本，或上传新的固件文件。

对话框中有一对单选按钮，用于指定是上传还是下载固件。 在第一种情况下，用户必须选择一个固件文件。 在第二种情况下，用户必须输入固件的输出文件名。
接下来有一个带有按钮的栏目

  • 帮助
  • 关闭 对话框而不执行任何操作
  • 确定 确认上传或下载固件

在上传新固件之前，TopoDroid 会检查它是否与硬件版本兼容。 不幸的是，没有确切的方法从文件内容中读取固件版本； 为了防止上传错误的文件， 会将文件中的签名块（在引导程序块之后的第一个块）与已知值进行比较，以及文件校验和进行比较。

检查结果代码如下（失败时为负数）：

  • 2100：固件 2.1
  • 2200：固件 2.2
  • 2300：固件 2.3
  • 2400：固件 2.4
  • 2412：固件 2.4c
  • 2500：固件 2.5
  • 2501：固件 2.51
  • 2512：固件 2.5c
  • 2610：固件 2.6.1
  • 2630：固件 2.6.3
  • 2640：固件 2.6.4
  • 2700：固件 2.7.0
  • -200：失败的 2.1 或 2.2
  • -230、-2300：失败的 2.3
  • -240、-2400：失败的 2.4
  • -250：失败的 2.5 或 2.51
  • -246、-2412：失败的 2.4c
  • -256、-2512：失败的 2.5c
  • -99：通用失败
DistoX（版本 2 或 BLE 版本）必须以引导程序模式启动。有关详细信息，请参阅 DistoX2 文档。
设备必须在引导程序模式下打开，因为 在上传新固件之前，TopoDroid 会从已安装在 DistoX 上的固件中读取签名块。 如果硬件版本不兼容，会有警告。

请注意，TopoDroid 不会警告您在硬件版本 1.1 上上传版本 2.5 之前的固件。

在上传固件时，TopoDroid 会显示一条消息，显示已写入 DistoX 的字节数。 如果这些字节数小于固件文件大小，则上传失败。

固件更新过程

  1. 确保 DistoX 是 TopoDroid 中选定的活动 DistoX
  2. 打开固件对话框（"固件" 菜单）并选择要上传的固件文件
  3. 检查您的 DistoX 的硬件和固件版本：确保激光已关闭（CLR），然后按两次 FUNC 显示版本。 您也可以使用 TopoDroid 设备信息对话框检查 DistoX 的固件和硬件
